# RAG(Retrieval-Augmented Generation) 기초

RAG는 LLM이 답변을 생성하기 전에 외부 지식 저장소에서 관련 문서를 검색(Retrieval)하여
그 내용을 근거(컨텍스트)로 삼아 답변을 생성(Generation)하는 기법이다.

## 왜 RAG를 쓰는가

- LLM의 학습 데이터에 없는 사내 문서, 개인 노트, 최신 정보에 근거한 답변이 가능하다.
- 근거 문서를 함께 제시할 수 있어 환각(hallucination)을 줄이고 출처를 검증할 수 있다.
- 모델을 재학습(fine-tuning)하지 않고도 지식을 갱신할 수 있다. 문서를 다시 색인하면 끝이다.

## 기본 파이프라인

1. **수집(Ingestion)**: 원본 문서를 불러와 텍스트로 변환한다.
2. **청킹(Chunking)**: 긴 문서를 검색 가능한 작은 조각으로 나눈다.
3. **임베딩(Embedding)**: 각 청크를 벡터로 변환한다.
4. **색인(Indexing)**: 벡터를 벡터 스토어에 저장한다.
5. **검색(Retrieval)**: 질문을 같은 방식으로 임베딩하여 유사한 청크를 찾는다.
6. **생성(Generation)**: 검색된 청크를 프롬프트에 넣어 LLM이 답변을 만든다.

## 청킹이 중요한 이유

청크가 너무 크면 관련 없는 내용이 섞여 검색 정확도가 떨어지고,
너무 작으면 문맥이 끊겨 LLM이 이해하기 어렵다.
일반적으로 500~1000자 수준에서 시작해 실험으로 조정하며,
청크 간 overlap을 두어 경계에서 문맥이 잘리는 것을 완화한다.
