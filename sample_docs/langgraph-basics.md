# LangGraph 기초

LangGraph는 LLM 애플리케이션을 **상태(State)를 공유하는 노드들의 그래프**로 구성하는 프레임워크다.
단순한 체인(chain)과 달리 조건 분기, 루프, 재시도 같은 제어 흐름을 명시적으로 표현할 수 있다.

## 핵심 개념

### State
그래프 전체가 공유하는 데이터 구조. 보통 `TypedDict`로 정의한다.
각 노드는 State를 입력으로 받아, 변경할 필드만 담은 부분 딕셔너리를 반환한다.

### Node
State를 받아 작업을 수행하는 함수. LLM 호출, 검색, 데이터 가공 등 무엇이든 될 수 있다.

### Edge
노드 간의 연결. 고정 엣지(`add_edge`)는 항상 다음 노드로 진행하고,
조건부 엣지(`add_conditional_edges`)는 State를 검사하는 함수의 반환값에 따라
다음 노드를 동적으로 결정한다.

## 왜 그래프인가

에이전트형 워크플로우는 "검색 결과가 나쁘면 쿼리를 고쳐 다시 검색한다" 같은
분기와 루프가 필수적이다. 이런 흐름을 if/while로 짜면 상태 관리가 흩어지지만,
그래프로 선언하면 흐름 전체가 한눈에 보이고 시각화(Mermaid)도 자동으로 얻는다.

## 컴파일과 실행

`StateGraph`에 노드와 엣지를 등록한 뒤 `compile()`을 호출하면 실행 가능한 앱이 된다.
`invoke()`로 한 번에 실행하거나 `stream()`으로 노드별 진행 상황을 받아볼 수 있다.
