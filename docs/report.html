02LangGraph — self-corrective 그래프

단순 retrieve→generate가 아니라, 검색 결과를 LLM이 재평가(grade)하고 전부 무관하면 쿼리를 재작성해 재검색(최대 2회)합니다. 점선이 조건부 엣지입니다.

04트러블슈팅 여정

실제 개발 중 부딪힌 문제들 — 순서대로 해결했고, 각 교훈은 코드와 스터디 문서에 반영되어 있습니다.

  1. 1 Notion 신규 API — databases.query가 사라짐

    notion-client 3.x는 2025-09 API를 사용해 조회가 데이터 소스 단위로 바뀜. 해결: DB ID → 데이터 소스 ID 해석 단계 추가 (data_sources.query).

  2. 2 ID 복사 함정 — 연결됐는데도 8개 DB 전부 404

    URL에서 손으로 복사한 ID가 실제 데이터 소스 ID와 다른 값이었음. 해결: search API로 접근 가능한 DB를 자동 발견하는 NOTION_DATABASE_ID=auto 모드 구현 → ID 복사 자체를 없앰.

  3. 3 Windows 파일 잠금 — 재색인 시 PermissionError

    Streamlit이 Chroma 파일을 열고 있으면 폴더 삭제(rmtree)가 실패. 해결: 폴더 삭제 대신 컬렉션 리셋(reset_collection())으로 전환.

  4. 4 빈 문서 함정 — 검색은 되는데 답할 내용이 없음

    명언집·단어장류 DB는 내용이 본문이 아닌 속성(컬럼)에 저장되어 있었음. 해결: rich_text·select·date 등 속성을 - 속성명: 값 텍스트로 함께 추출.
